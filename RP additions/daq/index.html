Data acquisition

The DAQ application is ready.

Now you can run the control program.

More details about this application can be found at not yet available link.

Getting started

  • Connect a signal source or the OUT1 to the IN1 connector on the Red Pitaya board.
  • Additionally you can connect another signal to the IN2 connector.
  • Clone the not yet available git repo.
  • Edit the config to your needs (ip is required). To play around with the values, the mcpha application might proove useful.
  • Run the rpDAQ.py program.

Additional Information for developers

  • This is based upon the 20240204 release
  • The FPGA Image is the same as the one of the mcpha except the FIR Filter was removed.
  • The Server Application is the same as the one of the mcpha except an additional command (31, 0, n) was added which starts the daq thread.
  • The DAQ thread waits for the triggerflag and then sends the data, resets and starts the osci n times.
  • As the DAQ thread first waits for the triggerflag you need to reset and start the oscilloscope before running the DAQ for the first time.

MimoCoRB

  • The Client also offers an interface to MimoCoRB which offers a framework for further live analysis.
